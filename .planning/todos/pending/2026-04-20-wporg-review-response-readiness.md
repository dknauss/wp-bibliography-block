## Problem

The plugin is submitted to WordPress.org and awaiting reviewer feedback. Turnaround quality depends on having metadata, package artifacts, and patch process ready.

## Solution

Keep release artifacts reproducible, monitor review responses, and prepare rapid patch/release workflow for any requested changes.

## Acceptance targets

- ready-to-rebuild `bibliography-block.zip` artifact
- readme/header metadata stays consistent with reviewer guidance
- reviewer requests can be addressed and redeployed in one patch cycle
